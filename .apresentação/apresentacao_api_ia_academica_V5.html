O Problema

  • 01

    Volume de materiais

    Alunos acumulam PDFs de aulas, múltiplas disciplinas e prazos simultâneos na pós-graduação

  • 02

    Dúvidas custam tempo

    Esclarecer dúvidas exige releitura manual de documentos extensos sem uma forma rápida de consulta

  • 03

    Controle descentralizado

    Tarefas e datas de entrega anotadas de forma manual, sem sistema unificado acessível

  • A DAPIA endereçou isso com agentes de IA via Telegram. Este trabalho adiciona a camada de API — tornando os agentes reutilizáveis por qualquer sistema.

Validação e Tratamento de Erros

Cenário HTTP Implementação
Sem header X-API-Key ou chave errada 401 security.py → require_api_key()
Campo question ausente ou < 5 chars 422 Pydantic v2 automático
Pasta data/aulas/ ou data/tarefas/ vazia 404 FileNotFoundError capturado
Erro inesperado no servidor 500 global_exception_handler
Resposta válida de qualquer endpoint 200 Hermes Agent respondeu com sucesso

Regra: Detalhes técnicos de erros internos nunca são expostos ao cliente — o log registra, a resposta HTTP protege.

10 / 14

Segurança e Logs

Autenticação via X-API-Key

Todos os endpoints de IA exigem o header. Chave configurada via variável de ambiente — nunca hardcoded. Mesmo padrão da API da OpenAI.

Health check público

GET / sem autenticação — retorna {status: ok, version: 1.0.0} para monitoramento externo.

Sistema de logs duplo

Terminal (desenvolvimento) + arquivo logs/app.log (produção). Formato: timestamp | nível | módulo | mensagem

Nível configurável via .env

LOG_LEVEL aceita DEBUG, INFO, WARNING ou ERROR. Arquivo .env protegido pelo .gitignore — credenciais jamais vão ao repositório.
